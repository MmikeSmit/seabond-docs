# 仓位怎么计算

SeaBond 的跟单不是按目标钱包的绝对仓位金额 1:1 照抄，而是按比例执行。

当带单员开仓时，系统会先计算这笔开仓成本占带单员账户总余额的比例，再用同样的比例换算到你的跟单账户。

## 示例

| 项目 | 示例 |
| --- | --- |
| 带单员账户总余额 | 10,000 USDC |
| 带单员本次开仓成本 | 1,000 USDC |
| 开仓成本占比 | 10% |
| 你的跟单账户总投入 | 500 USDC |
| 你的跟单账户对应投入 | 约 50 USDC |

也就是说，跟单关注的是“这笔交易占账户的比例”，而不是目标钱包用了多少绝对金额。这样大资金钱包和小资金跟单账户之间可以保持相对接近的仓位结构。

## 总投入的作用

你在创建跟单时填写的 **总投入**，会作为该跟单账户的初始资金基础。后续跟单下单时，会围绕这个跟单账户的资金规模按比例执行。

如果你投入 500 USDC，系统不会因为带单员开了 10,000 USDC 的仓位就让你也开 10,000 USDC，而是会按照带单员的仓位比例换算到你的 500 USDC 跟单账户中。

## 小额仓位处理

交易所通常有最小下单名义要求。如果按比例计算后的目标仓位低于最小下单名义，系统可能会按最小名义执行，以保证订单能够成交。

{% hint style="warning" %}
如果你的跟单资金规模远小于带单员账户，部分小比例仓位可能会因为最小下单名义、滑点或成交价格差异而和带单员结果不完全一致。
{% endhint %}

## 哪些参数由系统处理

创建跟单时，用户只需要选择带单钱包、跟单模式和总投入。杠杆模式、仓位换算方式、是否跟随开仓和平仓等执行参数由系统统一处理，页面不会要求用户手动配置。

这样做的目的，是让跟单规则保持一致：用户主要控制“跟谁、用多少资金、用哪种同步方式”，具体订单执行由系统按规则完成。
